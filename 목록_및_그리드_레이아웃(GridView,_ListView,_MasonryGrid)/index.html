목록 및 그리드 레이아웃¶

Row, Column 등 기본 레이아웃 사용을 익히면 GridView, ListView, MasonryGrid 와 같은 위젯을 사용할 수 있습니다.

목록 및 그리드 레이아웃을 사용하면 많은 양의 콘텐츠를 화면에 제공할 수 있어 복잡한 레이아웃을 구현에 용이합니다.

  • GridView: 자식 위젯을 2차원 격자 형태로 배치합니다. 상품 카드나 갤러리 화면을 구성할 때 주로 사용합니다.
  • ListView: 1차원 스크롤 리스트를 제공합니다. 스크롤 컨테이너의 가장 기본적인 형태입니다.
  • MasonryGrid: 정해진 열 수에 맞춰 자식 위젯들을 Pinterest 스타일의 불규칙한 높이로 배치합니다. 불규칙한 그리드 구조를 만들 때 이상적입니다.

GridView의 속성¶

GridView는 자식 위젯들을 2차원 격자(Grid) 형태로 배치하는 스크롤 가능한 레이아웃 위젯입니다. 이미지 갤러리, 상품 목록, 카드 리스트와 같이 행과 열의 구조가 일정한 UI를 구성할 때 가장 효과적입니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
visible bool true false일 경우 렌더링 및 레이아웃에서 완전히 제외됩니다.
onSelected Action unset 아이템 선택 시 실행할 액션을 정의합니다. (Detect, StateTransition 등)
crossAxisType Enum count 열의 단위를 지정합니다. (count: 고정 개수 / extent: 고정 폭)
crossAxisCount int 1 crossAxisType이 count일 때의 열 개수입니다.
crossAxisExtent double — crossAxisType이 extent일 때의 각 열의 너비(Pixel)입니다.
childAspectRatio double 1.0 각 아이템의 가로/세로 비율입니다. (너비 ÷ 높이)
mainAxisSpacing double 0.0 스크롤 방향(행 사이)의 간격입니다.
crossAxisSpacing double 0.0 열과 열 사이의 간격입니다.
padding EdgeInsets 0.0 그리드 내부의 전체 여백을 설정합니다.
scrollDirection Enum vertical 스크롤 축을 설정합니다. (vertical / horizontal)
physics Enum unset 스크롤의 물리적 효과를 설정합니다. (bouncing, clamping 등)
controller Controller — 스크롤 위치를 제어하거나 감지하는 컨트롤러를 연결합니다.
shrinkWrap bool false 콘텐츠 크기에 맞춰 그리드 전체 높이를 축소할지 여부입니다.
itemCount int 0 렌더링할 총 아이템의 개수입니다.
clipBehavior Enum hardEdge 경계를 벗어나는 콘텐츠의 클리핑 방식을 설정합니다.

2. 세부 설명 및 활용 팁¶

  • 배치 및 크기 결정 (crossAxisType & AspectRatio)

    그리드의 열 구조와 각 아이템의 형태를 결정하는 핵심 속성입니다.

    • crossAxisType:
      • count: 화면 너비와 상관없이 **고정된 열의 개수**를 유지합니다. (crossAxisCount 사용)
      • extent: 각 열의 **최대 너비**를 지정하며, 화면 너비에 따라 열 개수가 자동으로 조절됩니다. (crossAxisExtent 사용)
    • childAspectRatio:
      • 아이템의 가로 대비 세로 비율을 지정합니다. 예: 1.0은 정사각형, 2.0은 가로가 두 배 긴 직사각형을 의미합니다.
    • 간격 및 여백 (Spacing & Padding)
    • mainAxisSpacing / crossAxisSpacing: 아이템들 사이의 '사이 간격'만 설정합니다.
    • padding: 그리드 바깥쪽 테두리와 아이템 사이의 전체적인 여백을 설정할 때 사용합니다.
    • 스크롤 및 물리 효과 (Scroll & Physics)
    • scrollDirection: 기본값은 세로(vertical) 스크롤이며, 가로형 리스트를 만들 때 horizontal로 변경합니다.
    • physics:
      • bouncing: iOS 특유의 튕기는 효과를 줍니다.
      • clamping: Android 스타일의 고정된 스크롤 효과를 줍니다.
      • neverScrollable: 스크롤 기능을 끄고 고정된 화면으로만 보여줄 때 사용합니다.
    • 상호작용 (onSelected)
    • 그리드의 개별 셀을 클릭했을 때의 동작을 정의합니다. Action을 통해 상세 페이지로 이동하거나, Detect를 사용하여 선택된 셀의 인덱스를 변수에 저장하는 등의 로직을 구현할 수 있습니다.
💡 **성능 최적화 팁** 1. 아이템 개수가 매우 많을 경우 `shrinkWrap`은 가능하면 `false`로 유지하세요. `true` 설정 시 모든 아이템을 한꺼번에 레이아웃하므로 성능 저하의 원인이 될 수 있습니다. 2. 스크롤 시 키보드가 자동으로 내려가게 하려면 `keyboardDismissBehavior`를 `onDrag`로 설정하세요. 3. 이미지 썸네일 그리드를 만들 때는 `childAspectRatio`를 `1.0`으로 설정하여 정사각형 격자를 만드는 것이 일반적입니다.

ListView의 속성¶

ListView는 가장 널리 사용되는 1차원 스크롤 레이아웃 위젯입니다. 자식 위젯들을 수직 또는 수평 방향으로 나열하며, 화면의 범위를 벗어나는 많은 양의 콘텐츠를 효율적으로 보여줄 때 사용합니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
visible bool true false일 경우 레이아웃에서 완전히 제외됩니다.
scrollDirection Enum vertical 스크롤 방향을 설정합니다. (vertical / horizontal)
reverse bool false 스크롤 방향을 반전시킵니다. (채팅 UI 등에서 활용)
itemCount int 0 리스트에 표시될 아이템의 총 개수입니다.
padding EdgeInsets 0.0 리스트 내부의 전체 여백을 지정합니다.
physics Enum unset 스크롤 물리 효과를 설정합니다. (bouncing, clamping 등)
shrinkWrap bool false 콘텐츠 크기에 맞춰 리스트 전체 크기를 축소할지 결정합니다.
controller Controller — 스크롤 위치를 제어하거나 감지하는 컨트롤러입니다.
addAutomaticKeepAlives bool true 스크롤 밖으로 나간 아이템의 상태 보존 여부를 결정합니다.
clipBehavior Enum hardEdge 리스트 경계를 벗어나는 콘텐츠의 클리핑 방식입니다.

2. 세부 설명 및 활용 팁¶

  • 스크롤 방향 및 반전 (scrollDirection & reverse)
    • scrollDirection: 기본은 세로 방향(vertical)이지만, 가로로 넘기는 카드 리스트 등을 만들 때는 horizontal로 설정합니다.
    • reverse: true로 설정하면 리스트가 아래에서 위로(또는 오른쪽에서 왼쪽으로) 쌓입니다. 최신 메시지가 아래에 표시되어야 하는 채팅 UI 구현 시 필수적입니다.
  • 스크롤 물리 효과 (physics)
    • 사용자가 스크롤 끝에 도달했을 때의 반응을 정의합니다.
      • bouncing: iOS 스타일로, 끝에서 탄력 있게 튕겨 나갑니다.
      • clamping: Android 스타일로, 끝에서 멈추며 글로우 효과가 나타납니다.
      • neverScrollable: 스크롤 기능을 차단하여 고정된 리스트로 사용합니다.
  • 리스트 크기 최적화 (shrinkWrap)
    • 기본적으로 ListView는 부모가 허용하는 최대 공간을 차지하려 합니다. 하지만 다른 스크롤 뷰 안에 리스트를 넣거나, 콘텐츠 양만큼만 높이를 차지하게 하려면 shrinkWrap을 true로 설정해야 합니다.
💡 **리스트 활용 팁** 1. **무한 스크롤 구현:** `itemCount`를 데이터 리스트의 길이와 연동하고, `controller`를 통해 스크롤 끝에 도달했음을 감지하여 데이터를 추가 로드할 수 있습니다. 2. **성능 관리:** 아이템 개수가 수천 개 이상일 경우 `addRepaintBoundaries`를 활용해 변경이 있는 아이템만 다시 그리도록 설정하는 것이 좋습니다.

MasonryGrid의 속성¶

MasonryGrid는 자식 위젯들의 높이가 서로 다르더라도 빈 공간을 최적화하여 배치하는 Pinterest 스타일의 레이아웃 위젯입니다.

일반적인 GridView와 달리 행의 높이가 고정되지 않아, 불규칙한 크기의 이미지나 카드를 배치할 때 시각적으로 풍성한 UI를 제공합니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
crossAxisCount int 2 그리드의 열(Column) 개수를 지정합니다.
mainAxisSpacing double 0.0 수직 방향의 아이템 간 간격입니다.
crossAxisSpacing double 0.0 수평 방향의 아이템 간 간격입니다.
padding EdgeInsets 0.0 그리드 전체의 외부 여백을 설정합니다.
shrinkWrap bool false 자식들의 크기에 맞춰 그리드 전체 높이를 맞출지 여부입니다.
physics Enum unset 스크롤의 물리적 특성을 설정합니다.

2. 세부 설명 및 활용 팁¶

  • 열 개수 설정 (crossAxisCount)
    • MasonryGrid의 핵심은 crossAxisCount입니다. 화면을 몇 개의 열로 나눌지 결정하며, 각 아이템은 자신이 속한 열의 너비에 맞춰지고 높이는 콘텐츠에 따라 가변적으로 결정됩니다.
  • 간격 조절 (Spacing)
    • mainAxisSpacing: 아이템의 위아래 간격을 조절합니다.
    • crossAxisSpacing: 좌우 열 사이의 간격을 조절합니다.
    • 두 값을 적절히 활용하면 불규칙한 배치 속에서도 정돈된 느낌의 그리드를 만들 수 있습니다.
💡 **MasonryGrid 활용 팁** 1. **이미지 갤러리:** 사진의 원본 비율(가로형, 세로형)을 그대로 살리면서 꽉 찬 화면을 구성하고 싶을 때 `GridView`보다 훨씬 뛰어난 결과물을 보여줍니다. 2. **콘텐츠 카드:** 텍스트 양에 따라 높이가 달라지는 피드(Feed)형 UI를 구성할 때 빈 공간(Gutter)을 최소화하여 효율적인 화면 구성을 도와줍니다.
