콘텐츠 위젯이란¶

**콘텐츠 위젯**은 루시스튜디오에서 애플리케이션의 화면을 구성하는 가장 기본적인 단위이자 핵심 구성 요소입니다. 캔버스 위에 배치되어 텍스트, 이미지, 아이콘 등 실질적인 정보를 화면에 시각화하고 사용자와의 상호작용을 담당합니다.

콘텐츠 위젯 종류¶

콘텐츠 위젯은 목적에 따라 다양한 카테고리로 나뉩니다. 각 위젯의 특징과 종류는 다음과 같습니다.

분류 (Category) 제작 목적 위젯 종류
기본 정보 표시 텍스트, 이미지 등 기본 정보를 보여줄 때 Text, RichText, AttrText, Image, SVG, Lottie, Icon, QR, Badge
사용자 입력 및 선택 사용자가 값을 입력하거나 선택하게 할 때 TextField, Button, Checkbox, Radio, ImageRadio, DropDown, Switch, Slider, RangeSlider, Sign, DatePicker, Timer
미디어 및 서비스 연동 영상, 지도, 외부 기능을 연결할 때 Video, Youtube, Audio, Map, WebView, Camera, Scanner, HighChart
리스트 및 레이아웃 데이터 목록이나 화면 구성을 관리할 때 ListTile, Calendar, TabBar, ButtonTabBar, Form, ProgressBar
재사용 및 확장 시스템 구성이나 직접 제작한 요소를 쓸 때 Component, CustomWidget

[위젯 공통 용어 미리 보기]

용어 설명
Layout ID Lucy Studio가 위젯을 구별하기 위해 부여하는 고유ID 입니다.
스크립트에서 myText.text = 'hello' 처럼 참조할 때 필수입니다.
Property 탭 선택한 위젯의 시각/동작 속성을 설정하는 공간입니다.
색상·폰트·Border 등 대부분의 커스터마이징이 여기서 이뤄집니다.
Data Binding 탭 외부 데이터(Source/Provider) 값을 위젯 속성에 바인딩할 때 사용합니다.

1. 기본 정보 표시¶

Text, Image, Icon 등 앱 화면의 시각적 기초 정보를 구성하는 위젯들입니다. 주로 데이터를 화면에 출력하거나 브랜드 아이덴티티를 표현하는 용도로 활용됩니다.

위젯 종류¶

위젯 설명 사용 팁
Text 단일 스타일의 문자열을 출력하는 가장 기본적인 위젯입니다. 라벨이나 짧은 문구에 적합합니다.
RichText 한 줄 안에서 부분별로 스타일(굵기, 색상, 링크 등)을 다르게 적용할 수 있습니다. 이용약관 동의문처럼 특정 단어 강조가 필요할 때 사용합니다.
AttrText 데이터 바인딩을 통해 값의 상태(상승/하락 등)에 따라 색상이 자동 변경되는 텍스트입니다. 주가나 잔액 표시처럼 동적인 데이터 시각화에 최적화되어 있습니다.
Image 사진(JPG, PNG) 등 래스터 이미지를 화면에 표시합니다. 해상도별 이미지(asset)나 외부 URL 이미지를 모두 지원합니다.
SVG 해상도 저하 없는 벡터 그래픽(SVG)을 표시하며, 단색 틴트 처리가 쉽습니다. 로고나 고정 컬러 아이콘 제작 시 권장합니다.
Lottie JSON 기반의 고품질 벡터 애니메이션을 구동합니다. lottiefiles.com의 파일을 활용해 인터랙티브한 UX를 제공할 수 있습니다.
Icon 시스템 기본 아이콘 또는 커스텀 폰트 아이콘을 표시합니다. 버튼화하거나 간단한 심볼을 표시할 때 가볍게 사용 가능합니다.
QR 입력된 텍스트나 URL을 기반으로 QR 코드를 자동 생성합니다. 스마트폰으로 빠르게 스캔할 수 있는 코드를 제공합니다.
Badge 다른 위젯 모서리에 알림 숫자나 점을 표시합니다. 읽지 않은 메시지 수 표시 등에 활용됩니다.

주요 위젯¶

  1. Text

    화면에 단일 스타일의 문자열을 출력하는 가장 기본적인 위젯입니다.

    속성 설명 및 팁
    text 실제로 표시될 문자열입니다. 바인딩이 없는 기본값일 경우 "Hello World"로 렌더링됩니다.
    style.color Lucy 팔레트(Theme Token) 또는 HEX 입력을 지원합니다. 투명도 100%(FF)를 기본값으로 사용하십시오.
    fontSize dp(logical pixel) 단위입니다. 반응형 기준이 필요하면 MediaQuery 대신 Lucy의 Responsive 패널 활용을 권장합니다.
    fontWeight Lucy 드롭다운 값이 Flutter의 FontWeight 열거형 이름과 1:1로 매핑되어 두께를 조절합니다.
    2. Image

    사진이나 그림 등 다양한 소스의 이미지를 화면에 표시합니다.

    속성 설명 및 팁
    imageProvider assetImage(앱 번들 에셋), networkImage(서버/클라우드 URL), fileImage(기기 로컬 경로) 등 4가지 모드를 지원합니다.
    assetName / URL 선택한 모드에 맞는 이미지 경로 또는 URL을 입력합니다.네트워크 이미지의 경우 반드시 https:// 포맷을 사용해야 합니다.
    placeholder 네트워크 이미지가 로딩되는 동안 임시로 표시할 로컬 에셋 경로를 지정할 수 있습니다.
    fit cover(크롭 채움), fill(늘리기), contain(영역 내 맞춤) 등 이미지가 영역 내에 배치되는 방식을 결정합니다.
    isAntiAlias 이미지의 회전이나 ClipRRect 변형을 연속 적용할 때 발생하는 계단 현상을 완화합니다. 일반적인 UI에서는 성능을 위해 **OFF(false)**를 권장합니다.
    3. AttrText

    데이터 바인딩된 값의 상태에 따라 텍스트의 스타일이나 색상을 동적으로 변경하는 위젯입니다.

    속성 설명 및 팁
    useColorByValue true 설정 시 데이터 값의 상태(양수/음수/보합)에 따라 텍스트 색상을 자동으로 변경합니다.
    bullColor 데이터 값이 전일 대비 상승(+)할 경우 적용되는 색상입니다.
    bearColor 데이터 값이 전일 대비 하락(-)할 경우 적용되는 색상입니다.
    deerColor 데이터 값이 변동이 없거나(0) 보합일 때 적용되는 색상입니다.
    reverse 활성화 시 설정된 텍스트 색상과 배경 색상이 서로 반전되어 강조 효과를 줍니다.
    text 초기 텍스트 값이며, 데이터가 바인딩되면 해당 값으로 자동 업데이트됩니다.

2. 사용자 입력 및 선택¶

사용자가 직접 값을 입력하거나 준비된 옵션 중 하나를 선택하여 시스템과 상호작용하는 위젯들입니다. 데이터 수집, 설정 변경, 액션 실행 등 앱의 동적인 기능을 구현하는 데 필수적인 요소들입니다.

위젯 종류¶

위젯 설명 사용 팁
TextField 텍스트를 입력받는 창으로, 키보드 타입 및 마스킹 설정을 지원합니다. 비밀번호 입력이나 검색창 구현 시 핵심적으로 사용됩니다.
Button 클릭 시 특정 액션이나 스크립트를 실행하는 위젯입니다. 페이지 이동, 데이터 저장 등 사용자 명령의 시작점입니다.
Checkbox 불리언(true/false) 형태의 선택값을 입력받습니다. 여러 항목 중 다중 선택이 필요한 경우에 적합합니다.
Radio 동일 그룹 내에서 하나의 항목만 선택할 수 있도록 제어합니다. 성별 선택과 같이 상호 배타적인 옵션 제공 시 사용합니다.
ImageRadio 이미지를 기반으로 한 라디오 또는 체크박스 형태의 버튼입니다. 텍스트보다 시각적인 선택지가 효과적일 때 권장합니다.
DropDown 클릭 시 펼쳐지는 리스트에서 항목 하나를 선택합니다. 화면 공간을 절약하면서 많은 선택지를 제공할 때 유용합니다.
Switch 온/오프 상태를 전환하는 토글 스위치 위젯입니다. 설정값의 활성화/비활성화를 직관적으로 나타냅니다.
Slider 트랙을 드래그하여 특정 범위 내의 단일 수치를 조절합니다. 볼륨 조절이나 밝기 설정 등 연속적인 값 변경에 적합합니다.
RangeSlider 두 개의 핸들을 이용해 수치의 하한선과 상한선 범위를 지정합니다. 가격 범위 필터링 등 범위 설정이 필요할 때 사용합니다.
Sign 값의 상태에 따라 상승/하락/보합 아이콘을 자동으로 표시합니다. 주가 변동이나 지수 변화를 시각화하는 데 특화되어 있습니다.
DatePicker 연도, 월, 일을 스크롤 방식으로 선택할 수 있는 휠 기반 위젯입니다 날짜 입력이 필요한 예약이나 프로필 설정에 사용됩니다.
Timer 타이머의 실행, 정지, 종료 이벤트를 제어하는 위젯입니다. 스크립트를 통해 타이머 동작을 정밀하게 관리할 수 있습니다.

주요 위젯¶

  1. TextField

    사용자로부터 텍스트 입력을 받는 위젯으로, 다양한 입력 형식과 스타일 설정을 지원합니다.

    속성 설명 및 팁
    text 초기 표시 문자열입니다. Data Binding을 통해 Provider와 텍스트를 양방향으로 동기화할 수 있습니다.
    keyboardType number, emailAddress, datetime 등 입력 유형에 맞는 키보드 타입을 지정합니다.
    obscureText 활성화 시 입력 내용을 마스킹 처리하는 비밀번호 모드입니다.
    border enabled, focused, error, disabled 등 4가지 상태별로 테두리 스타일을 개별 설정할 수 있습니다.
    onChanged 입력이 발생할 때마다 실시간으로 실행되는 콜백 함수입니다.

  2. Button

    클릭 이벤트를 통해 특정 액션을 실행하는 위젯입니다. 설정 조합에 따라 다양한 형태의 버튼을 구현할 수 있습니다.

    속성 설명 및 팁
    onClick 버튼 클릭 시 실행될 스크립트 함수나 액션을 지정합니다.
    fillColor 버튼의 배경색입니다. 투명하게 설정하면 윤곽선(Outlined) 혹은 글자(Text)만 있는 버튼으로 활용 가능합니다.
    radius 버튼 모서리의 반경입니다. 0이면 직사각형, 40 이상이면 알약 모양(Pill) 형태가 됩니다.
    iconDirection 아이콘 삽입 시 텍스트를 기준으로 상·하·좌·우 배치 방향을 결정합니다.
    3. DropDown

    제시된 목록 중 하나의 항목을 선택하는 위젯입니다. 인라인 방식과 시트 방식을 모두 지원합니다.

    속성 설명 및 팁
    items 드롭다운에 표시할 옵션 리스트입니다. 문자열 배열 또는 JSON 형식을 지원합니다.
    comboType 인라인으로 펼쳐지는 'combo' 모드와 하단 모달 형태로 나타나는 'sheet' 모드 중 선택합니다.
    value 현재 선택된 값입니다. Data Binding을 통해 사용자의 선택값을 실시간으로 동기화할 수 있습니다.
    onChanged 선택 항목이 변경될 때 액션을 수행하거나 페이지를 이동하도록 설정할 수 있습니다.

3. 미디어 및 서비스 연동¶

Video, Audio 등 미디어 콘텐츠를 연동하거나 WebView, Map 등 외부 시스템의 기능을 앱 내로 가져와 풍부한 콘텐츠와 확장성을 제공하는 위젯들입니다. 외부 서비스와의 연결이나 기기의 하드웨어 기능을 활용할 때 주로 사용됩니다.

위젯 종류¶

위젯 설명 사용 팁
Video 동영상 콘텐츠를 재생하는 위젯입니다. 서비스 내 가이드 영상이나 홍보 영상을 보여줄 때 사용합니다.
Youtube 유튜브 플랫폼의 영상을 앱 내에서 직접 재생합니다. 별도의 인코딩 없이 유튜브 URL만으로 간편하게 영상을 공유할 수 있습니다.
Audio 오디오 파일을 재생하는 위젯입니다. 배경음악이나 안내 음성 등을 제공할 때 활용됩니다.
Map 위치 정보와 지도를 화면에 표시합니다. 길 찾기 서비스나 특정 장소의 위치를 안내할 때 필수적입니다.
WebView 외부 웹 페이지를 앱 화면의 일부로 불러옵니다. 기존 웹 서비스를 앱 내부에서 그대로 보여줄 때 유용합니다.
Camera 기기의 카메라를 호출하여 사진이나 영상을 촬영합니다. 전·후면 카메라를 모두 지원하며, 프로필 사진 등록 등에 활용됩니다.
Scanner 기기의 카메라를 통해 QR 코드를 스캔합니다. 스캔된 결과값을 스크립트로 전달받아 후속 액션을 처리할 수 있습니다.
HighChart highcharts.com의 기능을 활용하여 동적인 차트를 삽입합니다. 데이터를 시각화하고 사용자가 상호작용할 수 있는 차트를 제공합니다.

주요 위젯¶

  1. WebView

    외부 웹 콘텐츠를 앱 내부에 직접 로드하여 보여주는 위젯입니다.

    속성 설명 및 팁
    type 콘텐츠 로드 방식을 url(웹 주소), assetpath(내부 경로), data(HTML 직접 입력) 중에서 선택합니다.
    value (url) 이동하고자 하는 웹 주소 전체를 입력합니다.
    value (assetpath) 프로젝트 내부에 저장된 HTML 파일의 절대 경로를 입력하여 로컬 콘텐츠를 불러옵니다.
    value (data) 렌더링할 HTML 문자열을 직접 입력하여 화면을 구성할 수 있습니다.
    2. HighChart

    highcharts.com의 라이브러리를 활용하여 인터랙티브한 차트를 삽입하고 데이터를 시각화합니다.

    • 샘플 차트 가져오기

      단계 설명 및 팁
      1. Setting 속성 패널의 "Setting" 버튼을 클릭하여 전용 설정 모달을 호출합니다.
      2. 차트 추가 Sample Chart에서 프리셋을 선택하거나 highcharts.com 데모 페이지의 코드를 복사하여 붙여넣을 수 있습니다.
      3. 변수 생성 차트와 연결할 데이터 소스를 정의하고 관련 변수(Variable)를 생성합니다.
      4. 저장하기 “Save” 버튼을 클릭하여 저장합니다.
      - 차트에 데이터 바인딩하기
      단계 설명 및 팁
      1. 데이터 소스 설정 차트에 연결할 외부 데이터 소스(Data Source)를 먼저 설정해야 합니다.
      2. 변수 생성 차트와 연동할 변수(Variable)를 생성하고, 데이터 유형(String, Double 등)과 리스트 여부를 지정합니다.
      3. 변수 이름 입력 생성한 변수의 이름을 차트 스크립트 내의 data 또는 categories 속성값으로 직접 입력하여 데이터를 바인딩합니다.
      3. Scanner

    기기의 카메라 기능을 활용하여 QR 코드를 인식하고 그 데이터를 처리합니다.

    기능 설명 및 팁
    스캔 기능 활성화 시 기기의 카메라를 통해 실시간으로 QR 코드를 탐색합니다.
    결과 처리 인식된 데이터는 스크립트로 전달되며, 이를 통해 페이지 이동이나 데이터 저장 등의 후속 로직을 실행할 수 있습니다.
    접근 권한 하드웨어를 직접 제어하므로 카메라 사용에 대한 시스템 접근 권한 승인이 필요합니다.

4. 리스트 및 레이아웃¶

대량의 데이터를 효율적으로 보여주는 리스트 구성 요소와 화면의 정보 계층을 분리하는 레이아웃 위젯입니다. 복잡한 정보를 체계적으로 정리하여 사용자에게 높은 가독성을 제공하는 데 목적이 있습니다.

위젯 종류¶

위젯 설명 사용 팁
ListTile 아이콘, 제목, 부제 슬롯을 가진 표준 목록 행 위젯입니다. 리스트 형태의 레이아웃을 구성할 때 제목, 부제목, 좌우 아이콘 등을 조합하여 구조화된 UI를 만들 수 있습니다.
Calendar 달력 뷰를 통해 일정을 관리하고 특정 날짜를 선택할 수 있게 합니다. 일정(Event)이 있는 날짜에 점이나 뱃지를 표시할 수 있습니다.
TabBar 탭 메뉴를 통해 한 화면 내에서 콘텐츠 영역을 전환합니다. 탭의 개수가 많을 경우 스크롤 허용(isScrollable) 옵션을 사용하세요.
ButtonTabBar 버튼 스타일의 탭으로 직관적인 선택 인터페이스를 제공합니다. radius 값을 조절하여 직사각형 또는 알약 모양(Pill)으로 변경 가능합니다.
Form 다른 페이지를 현재 페이지 내의 특정 영역으로 불러와 구조화합니다. 여러 하위 페이지를 하나의 부모 레이아웃에서 통합 관리할 때 필수적입니다.
ProgressBar 작업의 진행 상태를 원형 또는 선형 인디케이터로 시각화합니다. value 값을 지정하지 않으면 무한 로딩 상태로 작동합니다.

주요 위젯¶

  1. ListTile

    리스트 형태의 레이아웃을 구성할 때, 각 항목(Row)을 쉽고 정형화된 형태로 구현하기 위해 사용하는 위젯입니다.

    속성 설명 및 팁
    leading 항목의 왼쪽에 배치되는 아이콘, 아바타, 체크박스 등을 위한 슬롯입니다
    title 항목의 중심이 되는 제목 텍스트입니다.
    subtitle 제목 아래에 위치하는 회색조의 보조 설명 텍스트입니다.
    trailing 항목의 우측 끝에 배치되는 화살표, 스위치, 메뉴 버튼 등을 위한 슬롯입니다.
    dense 활성화 시 행 높이가 기본 56px에서 48px로 압축되어 더 밀도 있는 리스트를 구성할 수 있습니다.
    onTap 해당 항목을 터치했을 때 실행될 스크립트 함수나 액션을 지정합니다.
    2. Calendar

    달력 형태로 날짜 정보를 표시하고, 특정 날짜 선택 및 일정을 시각화하는 위젯입니다.

    속성 설명 및 팁
    calendarFormat month(월), twoWeeks(2주), week(주) 단위로 달력의 표시 형식을 전환할 수 있습니다.
    firstDay / lastDay 달력에서 선택 가능한 날짜의 시작과 끝 범위를 제한합니다.
    eventDate List 일정이 있는 날짜 데이터의 배열을 연결하여 해당 날짜에 점이나 뱃지를 표시합니다.
    onDateSelected 날짜를 클릭했을 때 선택된 날짜 정보를 기반으로 실행될 콜백 액션입니다.
    3. Form(FormContainer)

    부모 레이아웃 내부에 다른 하위 페이지들을 포함시켜 관리하는 구조화 위젯입니다.

    • 사용 방법

      단계 설명 및 팁
      1. 위젯 배치 Form 위젯을 레이아웃의 원하는 위치에 드래그 앤 드롭합니다.
      2. ID 설정 스크립트에서 해당 위젯을 식별하고 제어하기 위해 Property 패널에서 고유한 layout ID를 설정합니다.
      3. 자식 페이지 제작 Form 영역 내부에 표시될 새로운 페이지를 생성하고 원하는 디자인을 구성합니다.
      4. 스크립트 호출 부모 페이지의 스크립트에서 formId.loadForm("페이지 경로")를 호출하여 자식 페이지를 로드합니다.
      5. 실행 및 테스트 페이지 간 데이터 인식 및 동기화를 위해 자식 페이지를 먼저 실행한 후 부모 페이지를 실행하여 결과를 확인합니다.
      - 주의사항
      - FormContainer는 상위(부모) 역할을 수행하며, 위젯 내에 삽입된 모든 페이지는 하위(자식) 페이지로 분류됩니다.
      - 데이터 연동 시 부모 페이지는 자신과 직접 연결된 직계 자식 페이지만 인식할 수 있습니다.
      - 단, $rootForm.postMessage(key, value) 명령어를 사용하면 계층 구조에 상관없이 최상위 부모 Form으로 데이터를 즉시 전달할 수 있습니다.
      - 최초 실행 시에는 반드시 자식 페이지 → 부모 페이지 순서로 실행해야 합니다.
      - 이후에는 부모 페이지만 실행 하면 됩니다.

5. 재사용 및 확장¶

시스템의 구성 요소나 직접 제작한 요소를 관리하여 디자인 일관성을 유지하고 개발 효율을 높이는 위젯들입니다. 반복되는 UI 구조를 모듈화하거나 특수한 기능을 확장할 때 사용됩니다.

위젯 종류¶

위젯 설명 사용 팁
Component 액션을 포함한 재사용 가능한 UI 블록을 생성하여 여러 페이지에 삽입합니다. 메인 컴포넌트 수정 시 모든 인스턴스에 자동 반영되어 유지보수가 쉽습니다.
CustomWidget 시스템에서 제공하지 않는 직접 제작한 요소를 레이아웃에 배치합니다. 표준 위젯만으로 구현하기 어려운 특수 기능이나 레이아웃 확장에 활용합니다.

주요 위젯¶

  1. Component

    액션을 포함한 재사용 가능한 UI 블록을 생성하여 여러 페이지에 효율적으로 삽입할 수 있는 위젯입니다.

  2. 사용 방법

    단계 설명 및 팁
    1. 컴포넌트 배치 원하는 페이지에서 Content 영역의 Component 위젯을 드래그합니다.
    2. 이름 설정 속성 패널에서 Component -> id 값을 컴포넌트 이름으로 설정합니다.(예: test.component)
    - 주의사항
    - 특정 인스턴스에서 속성을 별도로 수정한 경우, 이후 기본 컴포넌트에서 해당 속성을 변경해도 해당 인스턴스에는 적용되지 않습니다.
    - 개별 인스턴스에서의 변경은 기본 컴포넌트에는 영향을 주지 않습니다.
