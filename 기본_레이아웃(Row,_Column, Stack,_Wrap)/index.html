기본 레이아웃¶

Lucy Studio는 정교한 인터페이스를 구성하기 위해 Row, Column, Stack, Wrap 과 같은 레이아웃 위젯을 제공합니다.

이러한 위젯을 사용하면 앱 내에서 기본 위젯의 배치를 제어할 수 있습니다.

또한 이러한 기능에 익숙해지면 GridView, ListView 와 같은 더욱 복잡한 구성을 만들 수 있습니다.

  • Row:  자식 요소들을 왼쪽에서 오른쪽으로 가로 정렬합니다. 레이블이 있는 아이콘이나 툴바의 버튼처럼 요소들을 나란히 배치해야 하는 레이아웃을 만들 때 유용합니다.
  • Column: 자식 요소들을 위에서 아래로 세로 정렬합니다. 항목 목록을 만들거나 페이지의 특정 부분을 세로로 배치하는 데 적합합니다.
  • Stack: 자식 위젯들을 서로 겹쳐 배치할 수 있도록 해줍니다. 이미지 위에 텍스트를 배치하거나 아이콘에 배지를 추가하는 등 레이어 효과를 만드는 데 이상적입니다.
  • Wrap: 컨테이너의 크기가 자식 요소들보다 작아질 때, 콘텐츠를 자동으로 줄바꿈하여 다음 줄로 넘겨주는 기능입니다.

각 위젯은 서로 다른 목적을 가지고 있으며 어떤 위젯을 선택할지는 UI 구성 요소를 어떻게 배치해야 하는지에 따라 달라집니다.

Row, Column의 속성¶

1. 주축 크기(M**ain Axis Size)**¶

**레이아웃 위젯의 주축 크기**를 제어하는 속성입니다.

주축이란 레이아웃 위젯에 포함된 자식 요소들이 배치되는 주요 방향입니다.

예: Row 위젯의 주축은 가로 방향, Column 위젯의 주축은 세로 방향입니다.

  • Row 위젯의 주축 크기
    • 설명
      • Row 위젯이 가로 방향으로 차치할 너비를 결정합니다.
    • 옵션
      • min: 자식들의 합산 가로 너비만큼 차지합니다.
      • max: **부모가 허용하는 가능한 최대 너비**를 채웁니다. (기본값)
  • Column 위젯의 주축 크기
    • 설명
      • Column 위젯이 세로 방향으로 차치할 높이를 결정합니다.
    • 옵션
      • min: 자식들의 합산 세로 길이만큼 차지합니다.
      • max: **부모가 허용하는 가능한 최대 길이**를 채웁니다. (기본값)

2. 주축 정렬(M**ain Axis Alignment)**¶

자식 위젯들을 주축을 따라 정렬하는 속성입니다.

  • Row 위젯의 주축 정렬

    • 설명
      • 자식 위젯을 가로로 정렬하는 방식을 설정합니다.

    • 옵션

      • Start: 자식들을 왼쪽으로 정렬합니다.
      • Center: 자식들을 중앙으로 정렬합니다.
      • End: 자식들을 오른쪽으로 정렬합니다.
      • Space Between: 자식들 간의 간격이 동일하고, 양 극단에 여백을 남기지 않습니다.
      • Space Around: 자식들 간의 간격이 동일하지만, 자식들 사이 간격이 시작/끝 여백의 두배가 됩니다.
      • Space Evenly: 시작 여백, 끝 여백, 자식들 간의 간격이 모두 동일하게 정렬합니다.

  • Column 위젯의 주축 정렬

    • 설명
      • 자식 위젯을 세로로 정렬하는 방식을 설정합니다.

    • 옵션

      • Start: 자식들을 위로 정렬합니다.
      • Center: 자식들을 중앙으로 정렬합니다.
      • End: 자식들을 아래로 정렬합니다.
      • Space Between: 자식들 간의 간격이 동일하고, 양 극단에 여백을 남기지 않습니다.
      • Space Around: 자식들 간의 간격이 동일하지만, 자식들 사이 간격이 시작/끝 여백의 두배가 됩니다.
      • Space Evenly: 시작 여백, 끝 여백, 자식들 간의 간격이 모두 동일하게 정렬합니다.

3. 교차축 정렬(C**ross Axis Alignment)**¶

교차축은 **주축에 수직**이며, 이 보조 방향을 기준으로 위젯의 정렬 및 간격을 제어합니다. 교차축에는 시작, 중심, 끝의 세 가지 유형이 있습니다.

  • Row: 교차축은 세로 방향입니다. 자식 위젯을 위에서 아래로 정렬하는 방법을 결정합니다.

    • Start: 자식들을 위로 정렬합니다.
    • Center: 자식들을 세로 중앙으로 정렬합니다.
    • End: 자식들을 아래로 정렬합니다.
    • Stretch: 자식 위젯의 높이를 부모 Row 전체 높이만큼 늘립니다.
    • Baseline: 텍스트 기준선 정렬합니다. (사용 시 textBaseline 필요)
      • 서로 다른 폰트 크기의 텍스트가 한 줄에 섞여 있을 때 baseline으로 기준선을 맞춰 깔끔하게 정렬

  • Column: 주축은 세로 방향입니다. 자식 위젯을 왼쪽에서 오른쪽으로 정렬하는 방법을 결정합니다.

    • Start: 자식들을 왼쪽으로 정렬합니다.
    • Center: 자식들을 가로 중앙으로 정렬합니다.
    • End: 자식들을 오른쪽으로 정렬합니다.
    • Stretch: 자식 위젯의 너비를 부모 Column 전체 너비만큼 늘립니다.
    • Baseline: 텍스트 기준선 정렬합니다. (사용 시 textBaseline 필요)
      • 서로 다른 폰트 크기의 텍스트가 한 줄에 섞여 있을 때 baseline으로 기준선을 맞춰 깔끔하게 정렬

Stack, Wrap의 속성¶

Stack 속성¶

Stack 위젯은 자식들을 겹쳐서 배치하므로, 내부 요소들의 위치를 결정하는 정렬(Alignment) 속성이 중요합니다.

기본적으로 자식들을 좌측 상단부터 쌓아 올리며, Alignment 설정을 통해 중앙, 우측 하단 등 겹침의 기준점을 변경할 수 있습니다.

속성(Property) 정리

속성 타입 기본값 설명
alignment Alignment topLeft 위치가 지정되지 않은 자식 위젯들의 정렬 기준점을 설정합니다.
textDirection Enum ltr 자식 위젯들의 수평 배치 방향을 결정합니다.
fit Enum loose 위치가 지정되지 않은 자식 위젯들이 Stack의 크기에 어떻게 맞춰질지 결정합니다.
clipBehavior Enum hardEdge Stack의 범위를 벗어나는 자식 위젯의 렌더링 방식을 설정합니다.
**참고: StackFit 옵션** - **loose:** 자식 위젯이 Stack보다 작을 경우 자신의 크기를 유지합니다. - **expand:** 자식 위젯이 Stack의 전체 크기를 가득 채우도록 확장됩니다.

Wrap 속성¶

Wrap 위젯은 자식 위젯들을 가로 또는 세로로 나열하다가 공간이 부족하면 자동으로 다음 줄(또는 열)로 넘겨서 배치합니다.

버튼 목록이나 태그(Tag) 클라우드 구현에 적합합니다.

속성(Property) 정리

속성 타입 기본값 설명
direction Enum horizontal 자식 위젯을 나열할 주축 방향을 설정합니다.
alignment Enum start 주축(Main Axis) 방향의 자식 위젯 정렬 방식을 설정합니다.
runAlignment Enum start 여러 줄(Run)이 생겼을 때, 줄 사이의 전체 정렬 방식을 설정합니다.
spacing double 0.0 자식 위젯들 사이의 주축 방향 간격을 지정합니다.
runSpacing double 0.0 줄과 줄 사이(Run)의 간격을 지정합니다.
clipBehavior Enum none 범위를 벗어나는 자식 위젯의 처리 방식을 설정합니다.

공통 속성¶

TextDirection & VerticalDirection¶

start/end 정렬 키워드를 사용할 때 기준 좌우 방향을 결정합니다.

  • textDirection: 수평 방향의 흐름을 설정합니다.
    • ltr (Left-to-Right): 왼쪽에서 오른쪽으로 배치 (기본값)
    • rtl (Right-to-Left): 오른쪽에서 왼쪽으로 배치
  • verticalDirection: 수직 방향의 흐름을 설정합니다.
    • down: 위에서 아래로 배치 (기본값)
    • up: 아래에서 위로 배치 (역순)

ClipBehavior¶

부모 위젯의 경계(Boundary)를 벗어나는 자식 위젯의 콘텐츠를 어떻게 처리할지 결정합니다.

옵션 설명
none 영역을 벗어나도 자르지 않고 그대로 표시합니다. (성능 가장 좋음)
hardEdge 경계선에 맞춰 날카롭게 자릅니다. 안티앨리어싱(계단 현상 방지)이 적용되지 않습니다.
antiAlias 경계선을 부드럽게 자릅니다. 곡선이 포함된 레이아웃에서 주로 사용합니다.
antiAliasWithSaveLayer 가장 정밀하게 자르지만 성능 소모가 큽니다. 거의 사용되지 않습니다.
