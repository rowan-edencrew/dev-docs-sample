Lucy Script API Spec¶

이 문서는 **Lucy Script Engine**에서 JavaScript로 활용할 수 있는 내장 인터페이스 API의 상세 명세서입니다. 모든 API는 엔진에서 제공하는 전역 객체를 통해 호출할 수 있습니다.

1. Engine Module¶

앱의 핵심 기능을 제공하는 기본 모듈입니다.

1.1. $app (KAppManager)¶

앱의 전역 상태 관리 및 네비게이션을 담당하는 전역 객체입니다.

  • 변수명: $app
  • 전역 객체 여부: ✓ (Global)

[Properties - Getters]

플랫폼 정보 및 앱 설정을 조회할 수 있는 속성 리스트입니다.

Property Type Description
isAndroid boolean Android 플랫폼 여부 확인
isIOS boolean iOS 플랫폼 여부 확인
isWindows boolean Windows 플랫폼 여부 확인
isMacOS boolean macOS 플랫폼 여부 확인
isLinux boolean Linux 플랫폼 여부 확인
operatingSystem string 현재 구동 중인 OS 이름 반환
localeName string 설정된 로케일(Locale) 이름 반환
localHostname string 로컬 호스트 이름 반환
packageInfo object 앱 패키지 상세 정보 객체 반환

[Methods]

앱의 화면 이동 및 데이터 저장, 환경 설정을 제어하는 메서드 리스트입니다.

Method Return Type Parameters Description
openBackPage void - 이전 페이지로 이동
openPage void pagePath: string, linkArg: any, route?: string 지정된 경로의 페이지 열기
openPageReplacement void pagePath: string, linkArg: any, route?: string 현재 페이지를 새 페이지로 교체
openStudioConnector void - Studio Connector 화면 호출
setSharedData void sharedKey: string, data: any 런타임 공유 데이터 설정
getSharedData any sharedKey: string 설정된 공유 데이터 조회
setStoreData void key: string, data: any 영구 저장소(Store)에 데이터 기록
getStoreData any key: string 영구 저장소의 데이터 조회
removeStoreData void key: string 영구 저장소의 특정 데이터 삭제
getVar any appVarName: string 앱 전역 변수 값 조회
setVar void appVarName: string, data: any 앱 전역 변수 값 설정
clearVar void appVarName: string 앱 전역 변수 초기화
changeTheme void type: number 시스템 테마 변경
showLoadingIndicator void timeoutSeconds: number 로딩 인디케이터 표시
hideLoadingIndicator void - 로딩 인디케이터 숨김
isDarkTheme boolean - 현재 앱의 다크 테마 적용 여부
isDeviceDarkTheme boolean - 디바이스의 다크 테마 설정 여부

[Example]

1.2. $http (Http)¶

외부 서버와의 통신을 위해 HTTP 요청 기능을 제공하는 객체입니다.

  • 변수명: $http
  • 전역 객체 여부: ✓ (Global)

[Methods]

Method Return Type Parameters Description
get void url: string, jsCallback?: function GET 방식의 데이터 요청 수행
post void url: string, headers: object, data: string, jsCallback?: function POST 방식의 데이터 전송 및 요청 수행

[Example]

1.3. $log (LogManager)¶

애플리케이션 개발 시 로그 출력을 담당하는 객체입니다.

  • 변수명: $log
  • 전역 객체 여부: ✓ (Global)

[Methods]

Method Return Type Parameters Description
d void log: any Debug 레벨의 로그 출력
i void log: any Info 레벨의 로그 출력
w void log: any Warning 레벨의 로그 출력
e void log: any Error 레벨의 로그 출력

[Example]

1.4. $page (JetRunPage)¶

현재 페이지의 컨텍스트 내에서 변수 및 데이터를 관리하기 위한 객체입니다.

  • 변수명: $page
  • 전역 객체 여부: ✗ (현재 페이지 내에서만 유효)

[Methods]

Method Return Type Parameters Description
getVar any pageVarName: string 페이지 변수 조회
setVar void pageVarName: string, data: any 페이지 변수 설정
clearVar void pageVarName: string 페이지 변수 초기화
setSharedData void sharedKey: string, data: any 페이지 내 공유 데이터 설정
getSharedData any sharedKey: string 페이지 내 공유 데이터 조회
setStoreData void storeKey: string, data: any 페이지 내 저장 데이터(Store) 설정
getStoreData any storeKey: string 페이지 내 저장 데이터(Store) 조회

1.5. $form (JetRunForm)¶

폼/화면의 네비게이션 제어 및 폼 간 메시지 전달을 담당하는 객체입니다.

  • 변수명: $form, $parentForm, $ownerForm, $rootForm
  • 전역 객체 여부: ✗

[Methods]

Method Return Type Parameters Description
openPage void pagePath: string, linkArg: any, route: string 페이지 열기
openBackPage void - 이전 페이지로 이동
openStartDrawer void formPath: string, linkArg: any, jsCallback?: function 시작(Start) Drawer 열기
openEndDrawer void formPath: string, linkArg: any, jsCallback?: function 끝(End) Drawer 열기
openSheet void formPath: string, linkArg: any, jsCallback?: function Bottom Sheet 열기
openModalSheet void formPath: string, linkArg: any, jsCallback?: function, left?: number, top?: number, width?: number, height?: number 모달 시트(Modal Sheet) 열기
close void result: any 현재 폼 닫기
openDialog void formPath: string, linkArg: any, jsCallback?: function 다이얼로그(Dialog) 열기
