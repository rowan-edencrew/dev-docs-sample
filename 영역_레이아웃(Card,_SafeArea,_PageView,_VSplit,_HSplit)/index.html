영역 레이아웃¶

Card, SafeArea 등을 사용하면 화면의 특정 영역을 구분하거나 기기의 시스템 인터페이스와의 간섭을 방지하고, 화면을 논리적으로 분할하여 정교한 UI를 구성할 수 있습니다.

  • Card: 머티리얼 디자인의 카드 컨테이너 역할을 합니다. 프로필 카드나 정보 패널 등 UI 블록의 기본 빌딩 블록으로 활용됩니다.
  • SafeArea: 기기의 노치, 내비게이션 바 등 시스템 UI와 콘텐츠가 겹치지 않도록 자동으로 패딩을 추가합니다. 노치나 홈 바 침범을 방지하는 역할을 합니다.
  • PageView: 페이지 단위로 스크롤 가능한 뷰를 제공합니다. 온보딩 화면에서 페이지를 스와이프하여 전환하는 UI를 만들 때 사용합니다.
  • VSplit: 화면이나 컨테이너 영역에 세로 방향(Vertical)으로 경계선을 두어 좌/우 영역으로 분할합니다.
  • HSplit: 화면이나 컨테이너 영역에 가로 방향(Horizontal)으로 경계선을 두어 위/아래 영역으로 분할합니다.

Card의 속성¶

Card 위젯은 **머티리얼 디자인**의 카드 컨테이너 역할을 합니다.

배경색, 그림자, 모서리 곡률 등을 조절하여 콘텐츠를 입체감 있는 블록 형태로 시각화할 때 사용합니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
visible bool true false일 경우 렌더링 및 레이아웃에서 제외됩니다.
color Color #FFFFFF 카드의 배경색을 지정합니다.
shadowColor Color #000000 카드 그림자의 색상을 지정합니다.
elevation double 1.0 그림자의 높이(심도)를 설정합니다. 높을수록 더 높이 떠 있는 느낌을 줍니다.
shape Object - 카드의 형태(사각형, 둥근 사각형 등)와 테두리 스타일을 설정합니다.
margin EdgeInsets 4.0 카드 외부의 여백을 지정하여 다른 위젯과의 간격을 조절합니다.
clipBehavior Enum none 자식 위젯이 카드의 모서리 영역을 벗어날 때의 처리 방식을 결정합니다.
semanticContainer bool true 접근성 도구(스크린 리더 등)가 카드를 하나의 그룹으로 인식할지 여부입니다.

2. 세부 설명 및 활용 팁¶

  • 입체감 조절 (Elevation & Shadow): elevation 값을 조절하여 평면적인 디자인부터 팝업처럼 강조된 디자인까지 자유롭게 표현할 수 있습니다.
  • 모서리 곡률 (BorderRadius): shape 속성 내의 borderRadius를 활용해 모서리를 둥글게 깎을 수 있습니다. Lucy Studio에서는 전체 모서리(all) 또는 특정 위치(only)를 선택해 개별 설정이 가능합니다.
  • 테두리 오버레이 (BorderOnForeground): 테두리 선이 자식 콘텐츠 위에 그려지게 할지(true), 아래에 깔리게 할지(false) 결정하여 시각적 간섭을 제어합니다.

SafeArea의 속성¶

SafeArea는 최신 스마트폰의 노치(Notch), 상태 표시줄, 홈 인디케이터 등 시스템 UI와 콘텐츠가 겹치지 않도록 자동으로 안전한 여백(Padding)을 확보해 주는 필수 위젯입니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
visible bool true false 시 시스템 영역을 무시하고 전체 화면에 콘텐츠를 채웁니다.
left / top / right / bottom bool true 각 방향별로 안전 영역 패딩을 적용할지 여부를 선택합니다.
minimum EdgeInsets 0.0 시스템이 권장하는 기본 패딩 외에 추가로 유지해야 할 최소 여백입니다.
maintainBottomViewPadding bool true 소프트 키보드가 올라올 때 하단 시스템 패딩을 유지할지 여부입니다.

2. 세부 설명 및 활용 팁¶

  • 특정 방향 제외: 예를 들어, 배경 이미지를 화면 하단 끝까지 채우고 싶다면 bottom: false로 설정하여 하단 홈 인디케이터 영역까지 콘텐츠를 확장할 수 있습니다.
  • 전체 화면 앱: 동영상 플레이어나 게임처럼 시스템 UI 영역을 모두 사용해야 하는 경우 visible: false를 활용합니다.

PageView의 속성¶

PageView는 콘텐츠를 **페이지 단위**로 스와이프하여 전환할 수 있는 뷰를 제공합니다. 온보딩 가이드, 이미지 슬라이더, 전체 화면 전환 UI 구현에 적합합니다.

1. 속성(Property) 정리¶

속성 타입 기본값 설명
scrollDirection Enum horizontal 페이지가 넘어가는 방향을 설정합니다. (horizontal / vertical)
itemCount int 0 총 페이지의 개수를 지정합니다.
pageSnapping bool true 스와이프 시 다음 페이지에 딱 맞게 자석처럼 붙을지 결정합니다.
physics Enum unset 스크롤의 물리적 효과(바운스 등)를 결정합니다.
padEnds bool true 첫 번째와 마지막 페이지 끝에 여백을 두어 중앙 정렬을 유지할지 결정합니다.
allowImplicitScrolling bool false 인접한 페이지를 미리 렌더링(프리로딩)할지 여부입니다.

2. 세부 설명 및 활용 팁¶

  • 스와이프 방향: 수직(vertical)으로 설정하면 틱톡이나 인스타그램 릴스와 같은 상하 스와이프 방식의 UI를 손쉽게 구현할 수 있습니다.
  • 자동 스냅 (PageSnapping): 이 속성이 false이면 일반적인 리스트처럼 스와이프 도중 아무 곳에서나 멈출 수 있습니다. 페이지 단위 이동이 필요하다면 반드시 true로 유지하세요.
  • 상태 복원 (RestorationId): 앱이 백그라운드에서 종료되었다가 다시 실행될 때, 사용자가 마지막으로 보고 있던 페이지 번호를 기억해야 한다면 식별자(ID)를 지정하세요.
